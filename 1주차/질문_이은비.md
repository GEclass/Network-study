## 1. TCP/IP 계층이란 무엇이며, 각각의 계층에 대해서 설명해보라.

![image](https://github.com/GEclass/Network-study/assets/103405457/4cfd32e4-2394-48ba-ba18-b03f9b6a142f)

네트워크 시에 통신을 위해 사용하는 언어인 네트워크 아키텍처 중 TCP/IP를 이용하는 아키텍처를 TCP/IP 4 계층이라고 한다.

1. 네트워크 인터페이스 층
> 하나의 네트워크 내에서 L2 스위치가 0과 1의 디지털 데이터를 전기 신호와 같은 물리 신호로 변환해 전달한다. 즉, 같은 네트워크 내 인터페이스 간에 데이터를 전송한다.
- 같은 네트워크 내에서 데이터를 전송한다.
  - 이때, 하나의 네트워크란 라우터와 L3 스위치로 구획되는 범위 또는 L2 스위치로 구성되는 범위
2. 인터넷 층
- 많은 네트워크 사이에서 데이터를 전송한다.
  - 이때, 네트워트끼리 연결하고 데이터를 전송하는 라우터가 존재한다.
- 즉, 원격지 네트워크에서 출발지와 목적지 사이의 데이터 전송을 의미하는 엔드 투 엔드 통신을 한다.
3. 트랜스포트 층
- 데이터를 적절한 애플리케이션에 배분한다.
- 이때, TCP와 UDP를 이용하여 데이터를 배분한다.
4. 애플리케이션 층
> 문자, 음성, 동영상과 같은 0 또는 1의 디지털 데이터를 사용자가 이해할 수 있는 표현 형식으로 바꾼다.
> 또한, 클라이언트 - 서버 간에 데이터 형식과 처리 절차 등을 결정한다.
- 애플리케이션의 기능을 실행하기 위한 데이터의 형식과 처리 절차 등을 결정한다.

## 2. 클라우드 서비스의 분류에 대해서 설명하고, 각각의 분류에 대해 설명해보라.

네트워크를 경유해 서버의 어느 부분을 사용자가 이용할 수 있게 하냐의 따라 세 분류로 나뉨

1. IaaS(아이아스)
- 네트워크를 경유해 서버의 CPU와 메모리, 저장장치와 같은 하드웨어를 이용할 수 있게 함
2. PaaS(파스)
- 서버의 플랫폼(OS, 미들웨어)를 이용할 수 있게 함
3. SaaS(사스)
> 대부분의 클라우드 시스템
- 서버의 특정 소프트웨어 기능(애플리케이션)을 이용할 수 있게 함

### 미들웨어란 무엇인가? 예시를 통해 설명해보자.

미들웨어란 클라이언트와 DB 사이에 그 둘을 연결하는 시스템으로서 대표적으로는 이미지와 같은 정적인 데이터를 처리하는 웹 서버가 존재한다.

## 3. 네트워크를 구성하는 기기는 무엇이 있으며, 각각의 장비들은 무슨 역할을 하는가?

1. 라우터
2. 레이어2 스위치(L2 스위치)
3. 레이어3 스위치(L3 스위치)

위 대표적인 3개의 네트워크 기기는 모두 데이터를 전송한다.
이때, 데이터 수신 -> 데이터를 보낼 곳을 결정 -> 데이터 전송과 같이 데이터 전송을 처리한다.

1. 데이터 수신
- 물리적인 신호로 변환된 데이터를 0, 1과 같은 디지털 신호로 변환한다.
2. 데이터 보낼 곳을 결정
- 데이터에 부가된 제어정보(ex. 헤더)로 전송처를 결정한다.
3. 데이터 전송
- 물리적인 신호로 변환해서 데이터를 내보낸다.

### 스위치는 L2 스위치만 있을까?
OSI 7계층 기준, 스위치는 L2 ~ L7까지 존재하지만 주로 L2, L3 계층에서 동작하는 기기로 L2 스위치와 L3 스위치가 거론된다.

## 5. LAN vs WAN

1. LAN(Local Area Network)
- 사내 네트워크나 개인 사용자의 가정 내 네트워크를 구성
  - 사용자가 직접 구성하는 네트워크로서, 이더넷 인터페이스가 있는 라우터와 레이어2 및 레이어3 스위치를 준비한 뒤..
  - 네트워크 인터페이스를 LAN 케이블로 연결하면 LAN이 구성됨
 
2. WAN(Wide Area Network)
- 거점의 LAN끼리 연결한 네트워크

> A사 사내 네트워크
> LAN : 본사 네트워크, 인천 지사 네트워크, ..
> WAN : 거점(= 지사)의 LAN끼리 연결
 
### 이더넷이란?

LAN, WAN 등에서 가장 많이 활용되는 기술 규격으로 OSI 7계층 기준으로 다음과 같다.

1. 물리 계층 : 신호와 배선
2. 데이터 링크 계층 : MAC 패킷과 프로토콜의 형식을 정의

1. 캡슐화

> 헤더를 보고 프로토콜의 처리를 하고, 헤더를 제거한 후 다른 프로토콜에 처리를 넘김
- 데이터를 수신할 때, 헤더를 벗기는 과정 

## 7. 웹 브라우저에서 데이터를 보내는 과정을 설명해보자.

1. 웹 브라우저의 데이터는 우선 HTTP 헤더로 캡슐화되어 TCP로 넘어감(= 애플리케이션 층 -> 트랜스포트 층)
2. TCP 헤더가 추가되고
3. IP 헤더가 추가됨
4. 이더넷 헤더 및 FCS가 추가되면 네트워크로 내보낼 데이터 완성

따라서 이더넷의 규격에 대응하는 물리적인 신호로 데이터 전체를 변환해 전송 매체로 내보낸다.

### 반대로, 웹 브라우저에서 데이터를 수신하는 과정은? 

1. 웹 서버에 물리적인 신호가 도착하면 0 또는 1의 디지털 신호로 변환, 이후 이더넷 헤더 및 FCS를 제거하고 IP로 보냄
- 그리고 이더넷 헤더를 참조해 본인 앞으로 온 데이터인지 확인
- 또한, FCS를 통해 오류가 없는지 확인
2. 올바른 주소로 데이터가 전달 되었다면, IP 헤더를 제거한 후 TCP로 넘김
3. TCP 헤더를 참조해 어느 애플리케이션의 데이터인지 확인한 후 헤더를 제거. 그리고 HTTP로 넘김
4. HTTP 헤더 등을 제거하여 원본 데이터로 만듦

## 8. Ip vs Domain Name
### Ip 주소의 역할은 무엇인가?

IP(Internet Protocol)은 엔드 투 엔드 통신을 하는 것이다. 즉, 네트워크 상의 어떤 PC -> 다른 PC 등으로 데이터를 전송하는 것이 역할이다.
IP로 데이터를 전송하는 과정은 다음과 같다.

1. 데이터에 IP 헤더를 추가해 IP 패킷으로 만듦
- 이때, 출발지 및 목적지 IP의 주소가 반드시 지정되어야 함(= 통신 상대가 되는 호스트의 인터페이스 식별을 위해), 이 밖에 다른 정보들도 존재 
2. 라우터는 IP 헤더를 참조해 패킷을 전송한다.(= 라우팅)

## 9. 라우팅이란 무엇이며 왜 필요할까?
### 몇 계층에서 이뤄지는 작업인가?

OSI 7 계층 기준 - 3계층, TCP/IP 기준 - 2계층에서 이뤄지는 작업이다.
라우터는 네트워크를 연결하고, 데이터를 전송하기 행위를 하며(= 라우팅) 이는 반드시 필요한 작업이다.

## 10. 서브넷 마스크란 무엇이며 왜 사용하는가?

IP 주소는 전반의 네트워크부와 후반의 호스트부로 구성된다.
- 이때, 네트워크부와 호스트부의 구분은 고정된 것이 아니라 가변적

IP 주소는 총 32비트인데, 이때 어디까지가 네트워크부인지 명시한 것이 서브넷 마스크이다.
서브넷 마스크는 IP와 동일한 32비트로, 연속된 0과 1이 32개 나열된다. 또한, 이를 8비트씩 10진수로 변환하여 .으로 구분하여 표기한다.

이러한 서브넷 마스크는 IP 주소의 네트워크부와 호스트부를 구분한다.
- 호스트부가 모두 0이면 네트워크 주소
- 호스트부가 모두 1이면 브로드캐스트 주소
