# [Network] 1주차
---
## Q1. 네트워크의 종류는 무엇이 있는가?
  - 접속할 수 있는 사용자를 한정하는 사설 네트워크와 접속할 사용자를 제한하지 않고 누구든 이용할 수 있는 인터넷이 있습니다.
###  사설 네트워크에는 무엇이 있고 인터넷은 무엇인가?
  - 사설 네트워크에는 거점 내 기기를 연결한 LAN과, LAN끼리 연결한 거점 간 통신을 위한 WAN이 있습니다. 인터넷은  LAN, WAN을 연결한 전세계 규모의 네트워크입니다.
###  인터넷 구성 요소는 무엇이 있는가?
  - 인터넷은 전 세계의 다양한 조직이 관리하는 네트워크인 AS가 연결되어 있습니다. 대표적인 예로 인터넷 서비스 프로바이더, ISP가 있습니다.
---
## Q2. 클라우트 서비스는 어떻게 분류할 수 있는가?
  - 네트워크를 경유해 서버의 어느 부분을 사용자가 이용할 수 있게 하는가에 따라 하드워어 부분만 이용하는 IaaS, 네트워크를 경유해 OS/미들웨어까지 이용하는 PaaS, 하드웨어부터 애플리케이션 부분까지 SaaS가 있습니다.
---
## Q3. TCP/IP 계층에는 무엇이 있는가?
  - 애플리케이션의 기능을 실행하기 위한 데이터 형식과 절차를 결정하는 즉, 데이터를 문자와 이미지 등으로 표현하는 애플리케이션 층과 애플리케이션에서 데이터를 할당하며 데이터를 적절한 애플리케이션에 배분하는 트랜스포트 층, 엔드 투 엔드 통신을 하는 인터넷 층, 네트워크 안에서 데이터를 전송하는 네트워크 인터페이스 층이 있습니다.
###  TCP/IP 계층과 OSI 계층의 차이는 무엇인가?
  https://abangpa1ace.tistory.com/entry/Web-OSI-7%EA%B3%84%EC%B8%B5
  - OSI 7계층은 데이터 통신의 과정을 7개의 계층으로 분류한 모델이고, TCP/IP 4계층은 실제로 사용되는 프로토콜 스택입니다.
  - OSI 7계층에서의 2계층은 "데이터링크"를 의미하고, 3계층은 "네트워크"를 의미합니다.
  - OSI는 네트워크 구조를 명확하게 설명할 수 있는 개념적 모델이고, TCP/IP는 실제 인터넷 통신에 사용되고 있는 실전형 모델입니다.
  - TCP/IP의 애플리케이션 계층은 OSI의 애플리케이션, 세션, 프레젠테이션 계층을 통합한 포지션입니다. TCP/IP 모델의 인터넷 계층은 OSI 모델의 네트워크 계층에 대응하며, TCP/IP 모델의 네트워크 계층은 OSI의 데이터 링크, 물리 계층과 대응합니다.
  - OSI 모델이 먼저 개발된 다음 네트워크 아키텍처의 요구에 맞게 프로토콜이 만들어졌지만 TCP/IP 계층은 프로토콜이 먼저 개발되고 TCP/IP 모델이 구축됐습니다. 
###  계층별 데이터를 어떻게 부르는가?
  - 애플리케이션 층은 HTTP 헤더와 웹브라우저 데이터가 합쳐진 HTTP 메시지가 데이터입니다.
  - 트랜스포트 층은 TCP를 이용한다면, TCP 헤더와 HTTP 헤더와 웹 브라우저 데이터가 있는 세그먼트가 데이터이고 UDP를 이용한다면, 데이터 그램이 데이터입니다.
  - 인터넷 층은 IP 헤더와 TCP 헤더, HTTP 헤더, 웹 브라우저 데이터가 합쳐진 패킷이 데이터입니다.
  - 네트워크 인터페이스 층은 이더넷 헤더와 IP 헤더, TCP 헤더, HTTP 헤더, 웹 브라우저 데이터, FCS가 합쳐진 이더넷 프레임이 데이터입니다.
    - 데이터의 오류 유무를 확인하는 FCS

## Q4. 헤더란 무엇인가?
  - 각 프로토콜의 기능을 실현하기 위한 제어 정보입니다.
###  계층별로 어떤 헤더가 붙는가?
  - 애플리케이션 층은 HTTP 헤더가 붙습니다.
  - 트랜스포트 층은 TCP 헤더와 HTTP 헤더가 붙습니다.
  - 인터넷 층은 IP 헤더와 TCP 헤더, HTTP 헤더가 붙습니다.
  - 네트워크 인터페이스 층은 이더넷 헤더와 IP 헤더, TCP 헤더, HTTP 헤더가 붙습니다.

## Q5. IP란 무엇인가?
  - 엔드 투 엔드 통신을 하는 프로토콜입니다. 호스트 간 주고 받을 데이터에 IP 헤더를 추가하여 패킷으로 만듭니다.
###  IP 헤더에서 중요한 것은 무엇인가?
  - 출발지 IP 주소와 목적지 IP 주소가 중요합니다. 어떤 호스트가 출발이고 어떤 호스트가 도착인지에 따라 데이터를 전송하는 IP이므로 IP 주소가 제일 중요합니다. 
###  IP 주소는 어떻게 표기하는가?
  - 32비트를 8비트씩 10진수로 변환하여 표기하는 도트형 10진 표기를 사용합니다.
---
## Q6. 목적지 수에 따른 데이터 전송 방식에는 무엇이 있는가?
  - 단 한 곳으로 데이터를 전송하는 유니캐스트와, 같은 네트워크에서 모든 호스트에 완전히 똑같은 데이터를 전송하는 브로드캐스트, 특정 그룹에 포함되는 호스트에 완전히 똑같은 데이터를 전송송하는 멀티캐스트가 있습니다.
###  서브넷 마스크는 무엇인가?
  - IP 주소는 네트워크를 식별하는 네트워크부와 네트워크 내 호스트를 식별하는 호스트부로 이루어져 있습니다. IP 주소와 마찬가지로 32비트의 8비트씩 나누어서 1로만 이루어진 부분을 네트워크부로 0으로만 이루어진 부분을 호스트부로 구분하는 것이 서브넷 마스크입니다.
---
## Q7. 사설 IP 주소와 공인 IP 주소는 무엇인가?
  - 사설 IP 주소는 사설 네트워크에서 이용하는 IP 주소입니다. 공인 IP 주소는 전세계에서 유일한 IP로 ISP(인터넷 서비스 공급자)가 제공하는 IP 주소입니다. 
###  사설 네트워크에서 인터넷으로의 통신을 위해서는 무엇이 필요한가?
  - 인터넷에서 목적지가 사설 주소인 IP 패킷은 반드시 폐기되어 버립니다. 그래서 NAT 주소 변환을 통해 IP 패킷이 폐기되지 않게 해야 합니다. 요청을 전송할 경우, 사설 네트워크의 패킷 헤더에서 라우터가 출발 IP를 공안 주소로 변환하여 NAT 테이블에 보존하고 전송합니다. 요청을 응답할 경우, 라우터가 NAT 테이블에 보존한 사설 주소인 출발지 IP로 변환하여 응답을 사설 네트워크가 받아냅니다.
###  사설 IP 주소와 공인 IP 주소의 차이는 무엇인가?
  - 사설 IP 주소는 로컬 네트워크 통신을 목적으로 하며 공인 IP 주소는 인터넷 통신을 목적으로 합니다. 사설 IP 주소는 인터넷에서 식별할 수 없지만 공인 IP 주소는 인터넷에서 식별할 수 있으며 공인 IP 주소는 다른 IP 주소와 중복되지 않지만, 사설 IP 주소는 다른 네트워크 IP 주소와 중복될 수 있습니다.
---
## Q8. ICMP는 무엇인가?
  - IP에 의한 엔드 투 엔드 통신이 정상적으로 이루어졌는지 확인하는 프로토콜입니다.
###  ICMP의 기능은 무엇이 있는가?
  - IP 패킷을 폐기했다면 IP 패킷을 보낸 곳에 에러를 알리고 엔드 투 엔드 통신이 가능한지 `ping` 명령어로 진단할 수 있습니다. 
---
## Q9. ARP는 무엇인가?
  - IP 주소와 인터페이스를 식별하기 위한 MAC 주소를 대응시키는 프로토콜입니다.
---
## Q10. 포트번호는 무엇인가?
  - 호스트의 애플리케이션을 식별하는 것입니다.
###  포트 번호의 범위는 어떻게 되는가?
  - 0 ~ 1023은 서버 애플리케이션용으로 웰노운 포트이며, 1024 ~ 49151은 자주 이용되는 애플리케이션의 서버쪽 포트 번호입니다. 49152 ~ 65535는 클라이언트 애플리케이션용으로 동적/사설 포트입니다.
---
## Q11. TCP란 무엇인가?
  - 신뢰성 있는 애플리케이션 간의 데이터 전송을 하기 위한 프로토콜입니다.
###  TCP 프로토콜에서 보내는 단위는 무엇이며 어떻게 이루어져 있는가?
  - TCP 헤더와 HTTP 헤더, 애플리케이션 데이터가 합쳐진 TCP 세그먼트입니다.
###  TCP 헤더에서 중요한 것은 무엇인가?
  - 애플리케이션 간의 신뢰성있는 데이터 전송을 하는 TCP에서는 적절한 애플리케이션 프로토콜에 데이터를 배분할 수 있는 포트 번호가 중요합니다.
---
## Q12. TCP의 전송 절차는 어떻게 되는가?
  - TCP 커넥션 맺는 과정은
    1. 클라이언트가 서버에 접속을 요청하는 SYN 패킷을 보냅니다.
        - 클라이언트는 응답을 기다리는 SYN_SENT 상태가 됩니다.
    2. 서버는 SYN 패킷을 받고 요청을 수락한다는 ACK와 SYN flag가 설정된 패킷을 발송합니다.
        - 서버는 응답을 기다리는 SYN_RECEIVED 상태가 됩니다.
    3. 클라이언트는 서버에 ACK를 보냅니다.
        - 서버와 클라이언트는 ESTABLISHED 상태가 됩니다.
  - TCP 커넥션을 끊는 과정은
    1. 클라이언트가 연결을 종료하겠다는 FIN 플래그를 전송합니다.
        - 클라이언트는 서버가 확인했다는 신호를 기다리는 FIN-WAIT 상태가 됩니다.
    2. 서버가 FIN 플래그를 받고 확인했다는 ACK를 보냅니다.
        - 서버는 자신의 통신이 끝날 때까지 CLOSE_WAIT 상태가 됩니다.
    3. 서버가 연결을 종료할 준비가 되면 연결 해지를 위한 준비가 되었음을 알리기 위해 FIN 플래그를 전송합니다.
        - 서버는 LAST-ACK 상태가 됩니다.
    4. 클라이언트는 해지 준비가 되었다는 ACK를 서버에 보냅니다.
        - 클라이언트는 TIME-WAIT가 되며 일정 시간이 지나면 세션을 만료하고 연결을 종료시키며 CLOSED 상태로 변화합니다. 서버는 ACK를 받고 CLOSED 상태가 됩니다.

## Q13. UDP란 무엇인가?
  - 데이터를 적절한 애플리케이션에 배분하는 프로토콜입니다.
###  UDP에서 사용하는 데이터 단위는 무엇인가?
  - UDP 헤더와 애플리케이션 데이터가 합쳐진 UDP 데이터 그램입니다.
###  UDP의 장점은 무엇인가?
  - TCP와 다르게 연결 설정이 없고 혼잡 제어를 하지 않기 때문에 TCP보다 전송 속도가 빠르며 네트워크 부하가 적습니다.
---
## Q14. TCP와 UDP의 차이는 무엇인가?
  - TCP는 연결형 프로토콜이며 UDP는 비연결형 프로토콜입니다. TCP는 신뢰성있는 데이터 전송을 하며 데이터 재전송을 하지만 UDP는 비신뢰성 데이터 전송을 하며 데이터 재전송을 하지 않습니다. TCP는 유니캐스트 통신만을 하지만 UDP는 유니캐스트, 브로드캐스트, 멀티캐스트 통신을 합니다.
---
## Q15. DNS는 무엇인가?
  - 호스트 이름에 대응하는 IP 주소를 자동으로 구합니다.
###  DNS 서버를 어떠한 과정으로 사용하는가?
  1. 클라이언트가 URL을 DNS에 보냅니다.
  2. DNS가 URL에 해당하는 IP 주소를 루트 도메인부터 질의하여 클라이언트에게 보냅니다.
  3. 클라이언트가 DNS로 알아낸 IP 주소로 서버에 접속합니니다.
